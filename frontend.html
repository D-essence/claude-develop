UIコンポーネント概要

UIコンポーネントとは、Webページを構成する再利用可能な部品です。ユーザーインターフェースを一貫性のある、使いやすいものにするために、標準化されたデザインパターンに従って設計されています。

📝 UIコンポーネントを理解するメリット

UIコンポーネントの概念を理解すると、複雑なWebサイトも「部品の組み合わせ」として捉えることができ、Claudeにパーツごとに指示を出しやすくなります。

コンポーネント 主な用途 実装の難易度 レスポンシブ対応の重要度
ナビゲーションバー サイト内のメニュー表示、ページ間移動 中 非常に高い
カード 情報のグリッド表示、製品リスト 低 高い
フォーム ユーザー入力の収集 中〜高 高い
モーダル ポップアップ情報、確認ダイアログ 高 中
タブ・アコーディオン 関連コンテンツの整理、スペース節約 中 中
アラート・通知 ユーザーへのフィードバック 低 低
ヒーローセクション ページの主要メッセージの強調 中 高い
フッター 補助的なリンクや情報の表示 低 中

💡 効率的な開発のコツ

すべてのコンポーネントをゼロから作る必要はありません。Claudeにコンポーネントを依頼する際は、参考にしたいサイトや既存のライブラリ（BootstrapやTailwind UIなど）を指定すると、より高品質なコードが得られます。

📝 カードデザインのポイント

カードは情報の「島」を作るコンポーネントです。視覚的な区切りを作ることで、ユーザーが情報を整理して理解しやすくなります。シャドウ効果や微妙な境界線を使うことで、カードの「浮き上がり感」を表現できます。

Claudeさん、製品紹介用のカードコンポーネントのグリッドレイアウトを作成してください。 要件: 1. 3つの製品カードをグリッド表示（モバイルでは縦に積み重なる形に） 2. 各カードには画像領域、タイトル、説明文、価格、「購入する」ボタンを含める 3. カードにホバーした時の視覚効果も付ける（浮き上がる感じ） 4. 画像の代わりにプレースホルダーで構いません 5. カラーパレットは企業向け（青や紺を基調） 6. 角丸のモダンなデザイン HTMLとCSSのコードを提供してください。

フォーム

フォームはユーザーからの入力を収集するための重要なコンポーネントです。使いやすく、視覚的に分かりやすいフォームデザインは、コンバージョン率を高める鍵となります。

📝

入力フォーム

ユーザーから情報を収集するための様々な入力要素を含むコンポーネントです。連絡先フォーム、登録フォーム、ログインフォームなど多様な用途があります。

  • フォームはできるだけシンプルに、必要な項目だけに絞る
  • 入力フィールドには適切なラベルとプレースホルダーを付ける
  • エラーメッセージは具体的で分かりやすく
  • 必須項目と任意項目を明確に区別する
  • モバイルでの入力が快適になるよう十分な余白とサイズを確保
🔍

検索バー

ユーザーがサイト内のコンテンツを検索するための専用フォームです。シンプルさと視認性が重要です。

  • 検索アイコンや虫眼鏡マークを使って直感的に認識できるようにする
  • プレースホルダーテキストで検索対象や例を示す
  • 必要に応じてフィルターやソートオプションを追加
  • モバイルでは省スペース表示にしつつも、タップしやすい大きさを確保
Claudeさん、ユーザー登録用のフォームを作成してください。 要件: 1. 以下のフィールドを含めてください： - 氏名（姓・名を別々に） - メールアドレス - パスワード（確認用の再入力フィールドも） - 生年月日（セレクトボックスで年月日を選択） - 住所（郵便番号、都道府県、市区町村、番地など） - 利用規約への同意チェックボックス 2. 必須項目にはマークを付けて、わかりやすく 3. フォームバリデーション機能を含める（JavaScriptで入力チェック） 4. エラーメッセージは日本語で具体的に 5. レスポンシブデザイン対応 6. 送信ボタンは目立つデザイン HTMLとCSS、JavaScriptのコードを提供してください。

モーダル

モーダルウィンドウは、メインコンテンツの上に表示される小さなウィンドウで、ユーザーの注意を特定の内容に集中させるために使用されます。確認ダイアログ、詳細情報の表示、フォームの埋め込みなど様々な用途があります。

🪟

モーダルウィンドウ

ページの一部を暗くし、前面にポップアップ表示される小さなウィンドウです。ユーザーがモーダルを閉じるまで、背後のコンテンツとのインタラクションをブロックします。

モーダルのタイトル

モーダルの本文内容がここに表示されます。テキスト、画像、フォームなど、様々なコンテンツを含めることができます。

ユーザーはこのモーダルを閉じるまで、背後のコンテンツを操作できません。

  • モーダルは簡潔な内容に限定し、スクロールを必要としない程度のサイズにする
  • 閉じるボタンは明確に配置し、ESCキーでも閉じられるようにする
  • モーダル外側をクリックしても閉じられるようにすると使いやすい
  • モバイルでは画面いっぱいに表示されるよう調整する
  • フォーカストラップを実装して、モーダル内のフォーカスを保持する
🔔

タブ・アコーディオン

タブパネルとアコーディオンは、限られたスペースで複数のコンテンツセクションを整理して表示するための効果的なUIパターンです。関連する情報をグループ化し、ユーザーが必要な情報にアクセスしやすくします。

通知・アラート

通知やアラートは、ユーザーに重要な情報やフィードバックを提供するためのコンポーネントです。操作の結果、エラー、警告、ヒントなどを視覚的に伝えます。

レスポンシブデザイン

レスポンシブデザインは、様々な画面サイズやデバイス（デスクトップ、タブレット、スマートフォン）で適切に表示されるように設計する手法です。これは現代のWeb開発において必須のアプローチです。

レスポンシブデザインを実装するための基本的なテクニックを理解しましょう。これらを組み合わせることで、様々なデバイスに対応したサイトを作れます。

1. ビューポートの設定

これにより、モバイルブラウザがページを適切にレンダリングするための指示が与えられます。

2. メディアクエリの使用

CSSでスクリーンサイズに応じたスタイルを適用するための条件文です：

/* デスクトップ向けスタイル */ .container { display: grid; grid-template-columns: repeat(3, 1fr); gap: 20px; } /* タブレット向けスタイル */ @media (max-width: 991px) { .container { grid-template-columns: repeat(2, 1fr); } .sidebar { display: none; } } /* モバイル向けスタイル */ @media (max-width: 767px) { .container { grid-template-columns: 1fr; } .header { flex-direction: column; } }
3. フレキシブルグリッドの使用

固定サイズではなく、パーセンテージやフレックスボックス、グリッドレイアウトを使って柔軟なレイアウトを作ります：

.container { max-width: 1200px; width: 100%; margin: 0 auto; padding: 0 15px; } .flex-container { display: flex; flex-wrap: wrap; } .column { flex: 1 0 300px; /* 最小幅300pxで伸縮可能 */ }
4. フレキシブルイメージとメディア

コンテナからはみ出さないよう、画像やビデオを柔軟にサイズ調整します：

img, video, iframe { max-width: 100%; height: auto; }
5. モバイルファーストの設計

最初にモバイル用のスタイルを定義し、徐々に大きな画面サイズに対応するスタイルを追加していきます：

/* モバイル向け基本スタイル（メディアクエリなし） */ .nav { display: flex; flex-direction: column; } /* タブレット以上で水平メニューに変更 */ @media (min-width: 768px) { .nav { flex-direction: row; } }

📝 レスポンシブデザインの一般的なブレイクポイント

  • モバイル: 〜767px
  • タブレット: 768px〜991px
  • 小さいデスクトップ: 992px〜1199px
  • 大きいデスクトップ: 1200px〜

これらは一般的な値ですが、プロジェクトやデザインによって適切なブレイクポイントは変わります。デザインの崩れを基準にブレイクポイントを決めることが重要です。

API連携

APIとの連携は、Webアプリケーションが外部サービスやバックエンドサーバーと通信するための重要な要素です。JavaScript（特にFetch APIやAxios）を使って、データの取得や送信を行います。

API連携の仕組み

🔄

Fetch APIを使ったデータ取得

JavaScriptのFetch APIを使うと、外部APIやバックエンドサーバーからデータを取得できます。これはモダンなWeb開発の基本的なテクニックです。

// 基本的なGETリクエスト fetch('https://api.example.com/data') .then(response => { // レスポンスが正常かチェック if (!response.ok) { throw new Error('ネットワークレスポンスが正常ではありません'); } return response.json(); // JSONとしてパース }) .then(data => { console.log('取得したデータ:', data); // ここでデータを使った処理を行う（DOMの更新など） displayData(data); }) .catch(error => { console.error('データの取得に失敗しました:', error); showErrorMessage('データの読み込みに失敗しました。再度お試しください。'); }); // データをDOMに表示する関数 function displayData(data) { const container = document.getElementById('data-container'); container.innerHTML = ''; // 既存の内容をクリア // データを使ってHTML要素を生成 data.forEach(item => { const element = document.createElement('div'); element.className = 'item'; element.innerHTML = `

${item.title}

${item.description}

`; container.appendChild(element); }); } // エラーメッセージを表示する関数 function showErrorMessage(message) { const errorElement = document.getElementById('error-message'); errorElement.textContent = message; errorElement.style.display = 'block'; }
  • Fetch APIはPromiseベースで、非同期処理を簡潔に記述できます
  • 常にレスポンスが正常かどうかをチェックし、エラー処理を行いましょう
  • ユーザーに対して適切なローディング状態やエラーメッセージを表示しましょう
  • 本番環境では、クロスオリジンリソース共有（CORS）の問題に注意が必要です
📤

データの送信（POSTリクエスト）

フォームデータなどをサーバーに送信する場合、POSTリクエストを使用します。JSONデータをリクエストボディに含めて送信するのが一般的です。

// POSTリクエストの送信 async function submitFormData(formData) { try { // ローディング状態を表示 showLoadingIndicator(); const response = await fetch('https://api.example.com/users', { method: 'POST', // HTTP動詞を指定 headers: { 'Content-Type': 'application/json', // JSONデータを送ることを指定 // 必要に応じて認証トークンなども含める 'Authorization': 'Bearer ' + getUserToken() }, body: JSON.stringify(formData) // JavaScriptオブジェクトをJSON文字列に変換 }); // エラーチェック if (!response.ok) { throw new Error(`エラー: ${response.status} ${response.statusText}`); } // レスポンスをJSONとしてパース const result = await response.json(); // 成功メッセージを表示 showSuccessMessage('データが正常に送信されました'); return result; } catch (error) { console.error('データの送信に失敗しました:', error); showErrorMessage('送信に失敗しました。再度お試しください。'); throw error; // 呼び出し元でもエラーハンドリングできるようにする } finally { // ローディング表示を非表示にする hideLoadingIndicator(); } } // フォーム送信イベントハンドラーの例 document.getElementById('user-form').addEventListener('submit', async (event) => { event.preventDefault(); // デフォルトのフォーム送信を防止 // フォームからデータを収集 const formData = { name: document.getElementById('name').value, email: document.getElementById('email').value, message: document.getElementById('message').value }; try { const result = await submitFormData(formData); console.log('サーバーからの応答:', result); // 成功時の追加処理（リダイレクトなど） window.location.href = '/thank-you'; } catch (error) { // エラー時の追加処理 // すでにshowErrorMessageは呼ばれているので、ここでは何もしない } });
  • async/await構文を使うと、Promiseチェーンよりも読みやすいコードになります
  • try/catch/finallyでエラー処理とクリーンアップを適切に行いましょう
  • ユーザーにはローディング状態、成功、エラーの各状態を明確に示しましょう
  • 機密データの送信には必ずHTTPS（暗号化通信）を使用してください
🔄

サーバーレスAPI連携

Firebase、Supabase、AWS Amplifyなどのサービスを使用すると、自分でバックエンドサーバーを構築せずに、直接APIを使ってデータベースとやり取りできます。非エンジニアにとって特におすすめのアプローチです。

// Firebaseの例（Firestoreデータベース） import { initializeApp } from 'firebase/app'; import { getFirestore, collection, getDocs, addDoc, query, where } from 'firebase/firestore'; // Firebaseの初期化 const firebaseConfig = { apiKey: "YOUR_API_KEY", authDomain: "your-app.firebaseapp.com", projectId: "your-project-id", storageBucket: "your-app.appspot.com", messagingSenderId: "your-messaging-id", appId: "your-app-id" }; const app = initializeApp(firebaseConfig); const db = getFirestore(app); // データの取得 async function getUsers() { try { const usersCollection = collection(db, 'users'); const userSnapshot = await getDocs(usersCollection); const userList = userSnapshot.docs.map(doc => ({ id: doc.id, ...doc.data() })); return userList; } catch (error) { console.error("ユーザーの取得に失敗しました:", error); throw error; } } // フィルタリングを使用したクエリ async function getUsersByCity(city) { try { const usersCollection = collection(db, 'users'); const q = query(usersCollection, where("city", "==", city)); const querySnapshot = await getDocs(q); const userList = querySnapshot.docs.map(doc => ({ id: doc.id, ...doc.data() })); return userList; } catch (error) { console.error(`${city}のユーザー取得に失敗しました:`, error); throw error; } } // データの追加 async function addUser(userData) { try { const usersCollection = collection(db, 'users'); const docRef = await addDoc(usersCollection, userData); return { id: docRef.id, ...userData }; } catch (error) { console.error("ユーザーの追加に失敗しました:", error); throw error; } }
  • サーバーレスサービスを使うと、バックエンド開発の複雑さが大幅に軽減されます
  • 各サービスの認証機能、データベース、ストレージを活用できます
  • ただし、高度なカスタマイズやパフォーマンス最適化が必要な場合は、従来のバックエンド開発が適している場合もあります
  • APIキーなどの機密情報はクライアントサイドのコードに直接埋め込まないよう注意（環境変数などを使用）

効果的なプロンプトの書き方

Claudeから質の高いコードを生成してもらうには、プロンプトの書き方が重要です。具体的で明確な指示を与えることで、希望通りの結果が得られやすくなります。

1. 具体的な要件を明示する

良い例： 「3カラムのレスポンシブグリッドレイアウトを作成してください。モバイル（767px以下）では1カラム、タブレット（768px〜991px）では2カラム、デスクトップ（992px以上）では3カラムに変化するようにしてください。」

悪い例： 「グリッドレイアウトを作ってください。」

2. 希望するテクノロジーや手法を指定する

良い例： 「CSSフレックスボックスを使用して、中央揃えのナビゲーションメニューを作成してください。jQuery等のライブラリは使わず、純粋なJavaScriptだけで実装してください。」

悪い例： 「ナビゲーションメニューを作ってください。」

3. デザインの詳細や視覚的な要素を説明する

良い例： 「青系の配色（#4361ee をベースカラー）で、角が丸い（border-radius: 8px）カードコンポーネントを作成してください。カードにはホバー時に軽く浮き上がる効果（box-shadowとtransformを使用）を追加してください。」

悪い例： 「カードデザインを作ってください。」

4. 機能的な要件も詳細に説明する

良い例： 「クリックするとコンテンツが展開/折りたたみされるアコーディオンコンポーネントを作成してください。アニメーションはCSSトランジションで実装し、アクセシビリティのためにaria属性も適切に設定してください。一度に複数の項目を開けるようにしてください。」

悪い例： 「アコーディオンメニューが欲しいです。」

5. 参考にしたいサイトやデザインがあれば共有する

良い例： 「Bootstrap 5のナビゲーションバーのようなデザインで、ロゴが左側、メニュー項目が中央、ログインボタンが右側にあるレイアウトを作成してください。ただし、Bootstrapは使わず、純粋なHTML/CSS/JavaScriptで実装してください。」

6. 対応すべきエッジケースや考慮点を伝える

良い例： 「ユーザー登録フォームを作成してください。以下のバリデーションを含めてください：メールアドレスの形式チェック、パスワードの強度チェック（8文字以上、大文字小文字数字を含む）、フォーム送信前のすべてのフィールドの入力チェック。また、送信中の二重送信防止機能も追加してください。」
